Transformer Notes for RAG Chatbot

Overview
A Transformer is a neural network architecture designed for sequence modeling. It was introduced in the paper "Attention Is All You Need" and became the foundation of modern NLP systems such as BERT, GPT, T5, and many LLMs. The main idea of the Transformer is to use attention mechanisms instead of recurrence. Unlike RNNs and LSTMs, Transformers can process tokens in parallel, which makes training faster and more scalable.

Why Transformers Were Needed
Before Transformers, many NLP systems used RNNs, LSTMs, and GRUs. These models process sequences step by step. This sequential processing makes it difficult to parallelize training and can make long-range dependencies hard to learn. Transformers solve this problem by allowing every token to directly attend to other tokens in the sequence using self-attention.

Main Components of a Transformer
The original Transformer has an encoder-decoder architecture. The encoder reads the input sequence and creates contextual representations. The decoder generates the output sequence step by step. Modern models may use only the encoder, only the decoder, or both.

Encoder-only models are good for understanding tasks such as classification, named entity recognition, and extractive question answering. BERT and RoBERTa are examples of encoder-only Transformer models.

Decoder-only models are good for text generation because they predict the next token using previous tokens. GPT-style models are decoder-only Transformers.

Encoder-decoder models are good for sequence-to-sequence tasks such as translation, summarization, and text-to-text learning. T5 and BART are examples.

Tokens and Embeddings
Before text enters a Transformer, it is tokenized. Tokenization splits text into units such as words, subwords, or characters. Each token is converted into a vector using an embedding layer. These token embeddings represent the meaning of tokens as numbers.

Positional Encoding
Self-attention alone does not know word order. For example, it does not naturally know the difference between "dog bites man" and "man bites dog". To solve this, Transformers add positional information to token embeddings. Positional encoding tells the model where each token appears in the sequence.

The original Transformer used sinusoidal positional encoding. Some modern models use learned positional embeddings or relative positional encodings. Positional information is essential because language is ordered.

Self-Attention
Self-attention is the core idea of the Transformer. In self-attention, each token looks at other tokens in the same sequence and decides which tokens are important for understanding itself.

Example:
Sentence: "The animal did not cross the street because it was tired."
The token "it" should attend strongly to "animal" to understand what "it" refers to.

Self-attention allows the model to create contextual embeddings. This means the representation of a word changes depending on surrounding words. For example, "bank" in "river bank" and "bank account" will have different contextual representations.

Query, Key, and Value
Self-attention uses three vectors for each token: Query, Key, and Value.

Query represents what a token is looking for.
Key represents what a token offers or contains.
Value represents the information that will be passed forward if the token is attended to.

Attention compares Queries with Keys to compute attention scores. These scores are used to weight the Values. The final output is a weighted combination of Value vectors.

Scaled Dot-Product Attention
The Transformer computes attention using scaled dot-product attention.

Conceptually:
1. Compute similarity between Query and Key.
2. Scale the score to avoid very large values.
3. Apply softmax to convert scores into attention weights.
4. Use the attention weights to combine Value vectors.

The scaling factor is usually the square root of the key dimension. Scaling improves numerical stability.

Attention Weights
Attention weights show how much one token attends to another token. If a token has a high attention weight for another token, that other token strongly influences its representation.

Attention is useful because it helps the model focus on relevant parts of the input. It can capture dependencies between nearby and distant tokens.

Multi-Head Attention
Instead of using one attention mechanism, Transformers use multiple attention heads. Each head can learn a different type of relationship between tokens.

For example, one attention head may focus on grammar, another on semantic meaning, and another on long-distance relationships. The outputs of all heads are combined and projected into the final representation.

Multi-head attention makes the model more expressive because it can look at the sequence from multiple perspectives.

Masked Self-Attention
Decoder-only models such as GPT use masked self-attention. Masking prevents the model from looking at future tokens during generation.

For example, when predicting the next word in:
"I love machine ___"
the model can only use "I love machine", not future words.

Masked self-attention is necessary for autoregressive language modeling, where the model predicts the next token one step at a time.

Cross-Attention
In encoder-decoder Transformers, the decoder uses cross-attention to attend to the encoder output. The encoder represents the input sequence. The decoder uses this information while generating the output sequence.

Example:
Input: "Translate English to French: I love NLP"
The encoder processes the English sentence. The decoder uses cross-attention to generate the French output.

Feed-Forward Network
After attention, each Transformer block has a feed-forward neural network. This network is applied independently to each token representation. It adds non-linearity and helps transform the attended information into richer features.

Residual Connections
Transformers use residual connections around sublayers. A residual connection adds the input of a layer back to its output. This helps gradients flow during training and makes deep models easier to optimize.

Layer Normalization
Layer normalization stabilizes training by normalizing hidden representations. It helps the Transformer train more reliably and reduces internal instability. Modern Transformer variants may use different layer normalization placements, such as pre-layer normalization.

Dropout
Dropout is often used during training to reduce overfitting. It randomly drops some activations so the model does not depend too strongly on specific neurons.

Transformer Encoder
The Transformer encoder consists of repeated encoder blocks. Each encoder block usually contains:
1. Multi-head self-attention.
2. Residual connection and layer normalization.
3. Feed-forward network.
4. Residual connection and layer normalization.

The encoder creates contextual representations of all input tokens.

Transformer Decoder
The Transformer decoder consists of repeated decoder blocks. Each decoder block usually contains:
1. Masked multi-head self-attention.
2. Cross-attention over encoder outputs.
3. Feed-forward network.
4. Residual connections and layer normalization.

The decoder generates text one token at a time.

BERT
BERT is an encoder-only Transformer. It is designed mainly for language understanding. BERT is pretrained using masked language modeling, where some tokens are replaced with [MASK] and the model learns to predict them. Original BERT also used next sentence prediction.

BERT is useful for tasks such as sentiment classification, named entity recognition, and question answering. BERT is not a natural text generator like GPT.

RoBERTa
RoBERTa is an improved BERT-style model. It modifies the BERT pretraining strategy and removes next sentence prediction. It is often stronger than BERT for understanding tasks.

GPT
GPT models are decoder-only Transformers. They are trained to predict the next token. This makes them suitable for text generation, chatbots, summarization, code generation, and many LLM applications.

GPT uses masked self-attention so the model cannot see future tokens during training or generation.

T5
T5 stands for Text-To-Text Transfer Transformer. It converts every NLP task into a text-to-text format. For example, classification, translation, summarization, and question answering are all treated as text input to text output tasks.

T5 is an encoder-decoder Transformer.

BART
BART is an encoder-decoder Transformer designed for denoising and generation tasks. It is strong for summarization and text generation. BART combines ideas from BERT-style encoders and GPT-style decoders.

Transformers vs RNNs
RNNs process tokens sequentially, while Transformers can process tokens in parallel. RNNs carry information step by step through hidden states. Transformers use self-attention so each token can directly interact with other tokens.

Transformers are better suited for large-scale training and long-range dependencies, but their self-attention has quadratic complexity with sequence length.

Quadratic Complexity
Self-attention compares every token with every other token. If a sequence has n tokens, attention requires about n squared comparisons. This becomes expensive for very long documents.

This is why long-context Transformers and efficient attention methods are important research areas.

Transformers in RAG
RAG systems often use Transformers in two ways:
1. Embedding models convert documents and queries into vectors.
2. LLMs generate final answers using retrieved context.

A RAG chatbot may use a Sentence Transformer model for embeddings and a GPT-style or LLaMA-style model for answer generation.

Transformer Embeddings
Transformer-based embedding models can create contextual representations of text. Sentence Transformer models are often used to create sentence or paragraph embeddings for semantic search.

In RAG, document chunks and user questions are embedded into vector space. The retriever finds chunks whose embeddings are closest to the query embedding.

Context Window
Every Transformer-based LLM has a context window. The context window is the maximum amount of text the model can process at once. In RAG, retrieved chunks, chat history, and the user question must fit inside the context window.

If too much context is provided, the model may become slower, more expensive, or unable to process the input.

Why Transformers Matter for Chatbots
Modern chatbots are usually based on Transformer architectures. They can understand context, generate fluent text, follow instructions, and answer questions. LangChain and RAG systems do not replace the Transformer model; they organize how external data is retrieved and passed to the model.

Transformer Summary
A Transformer is a deep learning architecture based on attention. Its main strengths are parallel processing, contextual representations, multi-head attention, and scalability. It is the foundation of BERT, GPT, T5, BART, and many modern LLMs.
